## Завдання 1 — Рекурсивне сортування файлів (`task1.py`)

Скрипт рекурсивно копіює файли з вихідної директорії до директорії призначення,
розкладаючи їх у піддиректорії за розширенням файлу.

### Запуск

```bash
python task1.py <source_dir> [destination_dir]
```

* `source_dir` — шлях до вихідної директорії (обов'язковий).
* `destination_dir` — шлях до директорії призначення (необов'язковий, за замовчуванням `dist`).

### Приклад

```bash
python task1.py ./my_files ./sorted_files
```

Після виконання всі файли з `./my_files` (включно з вкладеними директоріями) будуть
скопійовані в `./sorted_files` і розсортовані у піддиректорії за розширенням:

```
sorted_files/
  txt/
    readme.txt
    notes.txt
  jpg/
    photo.jpg
  py/
    script.py
```

## Завдання 2 — Сніжинка Коха (`task2.py`)

Програма візуалізує фрактал «Сніжинка Коха» за допомогою рекурсії.
Користувач вводить рівень рекурсії перед запуском малювання.

### Запуск

```bash
python task2.py
```

Програма запитає рівень рекурсії (0 і більше):

```
Enter the recursion level (0 or more): 4
```

| Рівень | Опис |
|--------|------|
| 0 | Рівнобічний трикутник |
| 1 | Зірка Давида |
| 3–4 | Впізнавана сніжинка Коха |

## Завдання 3 — Порівняння алгоритмів сортування (`task3.py`)

### Запуск

```bash
python task3.py
```

### Алгоритми

| Алгоритм | Складність (гірший випадок) | Реалізація |
|----------|-----------------------------|------------|
| Сортування злиттям | O(n log n) | `merge_sort()` |
| Сортування вставками | O(n²) | `insertion_sort()` |
| Timsort (вбудований) | O(n log n) | `sorted()` / `list.sort()` |

### Результати вимірювань (середній час виконання, секунди)

| Розмір масиву | Сортування злиттям | Сортування вставками | Timsort |
|:-------------:|-------------------:|---------------------:|--------:|
| 100           | 0.000134           | 0.000105             | 0.000004 |
| 1 000         | 0.001258           | 0.013454             | 0.000072 |
| 5 000         | 0.007479           | 0.378023             | 0.000525 |
| 10 000        | 0.015891           | 1.520935             | 0.001133 |

*(Вимірювання проводились із використанням модуля `timeit`, 5 повторень, результат — середнє значення.)*

### Висновки

1. **Сортування вставками** демонструє O(n²) поведінку: при збільшенні масиву з 1 000 до 10 000
   елементів час зростає приблизно у 113 разів (замість очікуваних 10 для лінійного зростання).
   Алгоритм підходить лише для малих масивів або майже відсортованих даних.

2. **Сортування злиттям** має стабільну O(n log n) складність. Час зростає приблизно у 12,6 рази
   при збільшенні масиву у 10 разів (з 1 000 до 10 000). Проте чиста Python-реалізація
   несе значні накладні витрати через рекурсію та виділення пам'яті для нових списків.

3. **Timsort** (вбудований `sorted()`) є найшвидшим у всіх сценаріях — приблизно в **10–14 разів**
   швидший за власну реалізацію сортування злиттям і до **1 340 разів** швидший за сортування
   вставками (при n = 10 000). Це пояснюється:
   - реалізацією на мові C з мінімальними накладними витратами;
   - поєднанням переваг сортування злиттям (O(n log n)) та сортування вставками
     (ефективне для малих або вже впорядкованих підмасивів);
   - оптимізацією для реальних даних, які часто містять вже впорядковані «пробіги» (runs).

**Загальний висновок**: вбудовані алгоритми сортування Python практично завжди є кращим
вибором, ніж власна реалізація. Timsort поєднує теоретичну ефективність (O(n log n))
з практичними оптимізаціями, що підтверджується емпіричними даними.
